📋 Visão Geral do Projeto

Projeto profissional de automação QA para a plataforma ParaBank, desenvolvido seguindo as melhores práticas de engenharia de testes. A suíte inclui 50 casos de teste cobrindo funcionalidades críticas, segurança, performance e usabilidade.

⭐ Destaques do Projeto

  • Page Object Model (POM) - Arquitetura escalável e mantível com separação clara de páginas e testes
  • 50 Casos de Teste - Cobertura abrangente (funcionalidade, segurança, performance, usabilidade)
  • Data Factory - Geração automática de dados únicos impossíveis de duplicar
  • Test Helpers - Funções reutilizáveis para operações comuns
  • Type Safety - 100% TypeScript com tipos definidos
  • Relatórios HTML - Screenshots, vídeos e traces em caso de falha
  • Test Tagging - Organização por categorias (@smoke, @regression, @e2e, @security, @performance)
  • Performance Monitoring - Validação de tempo de resposta e carregamento
  • Security Testing - Testes de SQL Injection, XSS e HTTPS
  • CI/CD Ready - Fácil integração com pipelines de automação

🏆 Padrões & Boas Práticas

Page Object Model

Separação clara entre elementos de página e lógica de teste, facilitando manutenção e escalabilidade.

Data Factory Pattern

Geração automática de dados únicos e válidos para cada execução, evitando conflitos de dados.

Type Safety

100% TypeScript com interfaces definidas para garantir segurança de tipos em toda a suíte.

Test Tagging

Organização de testes por categorias permitindo execução seletiva (@smoke, @regression, etc).

Try-Catch Pattern

Testes resilientes que não quebram a suíte inteira mesmo com falhas individuais.

Constants Centralizados

URLs, seletores e dados centralizados para fácil manutenção e mudanças globais.
